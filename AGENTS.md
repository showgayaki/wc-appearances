# appearance-notice-generator

このプロジェクトは、テレビ出演情報を公開表示し、投稿用テキストを生成するための軽量Webアプリです。

公開ページでは出演情報と生成テキストを誰でも閲覧できます。番組データの追加・編集・削除は、Supabase Authでログインした管理ユーザーだけが行います。

## 技術スタック

- React
- TypeScript
- Vite
- Supabase
- pnpm

## ルール

- React は関数コンポーネントのみ使用する
- TypeScript の strict mode を前提とする
- `any` は使用しない
- 過度な設計を避ける
- ライブラリは目的と保守性を説明できる場合に追加する
- 自前実装より安全性、保守性、開発効率が明確に上がる場合はライブラリを利用する
- 使い捨ての小さな処理や、既存機能で十分な場合は依存を増やさない
- シンプルで実用的なUIを優先する
- Promise チェーンより async/await を使用する
- パッケージ管理は pnpm を使用する

## 開発チェック

- コードを変更した後は、原則として `pnpm lint` と `pnpm build` を実行する
- deprecated API が検出された場合は、可能な限り推奨APIへ置き換える
- lintやbuildを実行できなかった場合は、その理由を回答に明記する

## アーキテクチャ

アーキテクチャやディレクトリ構成は、プロジェクト規模に応じて適切に判断すること。

ただし以下を重視する：

- 保守しやすいこと
- 小さなコンポーネントへ分割しやすいこと
- 機能追加しやすいこと
- シンプルで理解しやすい構成であること
- 複雑な状態管理を避け、React標準の `useState` / `useEffect` / `useMemo` を中心にすること

## Supabase

- Supabase を使用する
- 認証情報は環境変数で管理する
- 機密情報をハードコードしない
- Supabase Auth は email/password を使用する
- アプリ上のログインUIは ID/PASS 形式にする
- 入力されたIDが `VITE_LOGIN_ID` と一致した場合のみ、内部で `VITE_LOGIN_EMAIL` に変換してログインする
- パスワードはSupabase Auth側で管理し、フロントの環境変数には置かない
- 初期ユーザー作成、メール確認、パスワード変更はSupabase管理画面で行う

必要な環境変数：

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`
- `VITE_LOGIN_ID`
- `VITE_LOGIN_EMAIL`

## 公開・権限方針

- 基本ページは公開する
- 未ログインでも、生成テキストと出演データ一覧を閲覧できる
- レギュラー番組、ゲスト・特番、投稿見出しの追加・編集・削除はログイン必須にする
- Supabase RLSは「読み取りは公開、書き込みはログイン済みのみ」にする
- 公開共有URLや複数ユーザー権限管理は現時点では不要

## データモデル

### `regular_programs`

レギュラー番組を管理する。

- 曜日
- 開始時刻
- 終了時刻
- 局
- 番組名
- 有効/無効

指定期間内の日付へ自動展開して、生成テキストに含める。

### `extra_programs`

レギュラー番組の自動展開とは別に追加する番組を管理する。

- 日付
- 開始時刻
- 終了時刻
- 局
- 番組名

ゲスト出演、特番、スペシャル放送などを保存・再利用するため、このテーブルは必要。

### `post_headers`

投稿文の見出し文言を管理する。

- 見出し

例：`🌈今週テレビ🌈`

登録済みの見出しがあれば使用し、なければデフォルト文言を使用する。

## テキスト生成仕様

生成された出演情報テキストは textarea に出力すること。

ユーザーが直接内容を編集・手直しできるUIにする。

完成済みテキストを固定表示するのではなく、最終調整しやすいことを重視する。

生成条件：

- 開始日・終了日はユーザーが手動指定する
- 対象期間内のレギュラー番組を曜日に応じて自動展開する
- 対象期間内のゲスト・特番を含める
- レギュラー番組とゲスト・特番を日時昇順で並べる
- 先頭には登録済みの投稿見出しを出す
- 投稿見出しがない場合はデフォルト文言を使う

時刻表記：

- `18:30` のような通常時刻を扱う
- `24:45` や `26:20` のような深夜表記も、そのまま入力・出力できるようにする
- 深夜表記を扱うため、時刻入力は必要に応じてテキスト入力を使う

## テキスト出力例

以下は形式確認用のダミー例。

実在の投稿文、日時、番組名、局をそのまま記載しないこと。

```text
🌈今週テレビ🌈

5/1(月)18:30〜19:00
サンプルテレビ「サンプル番組A」

5/2(火)24:45〜25:15
テスト放送「深夜のサンプル番組」

5/4(木)8:00〜9:55
デモテレビ「朝の情報番組」

5/5(金)26:20〜26:45
サンプル放送「深夜テストバラエティ」

5/7(日)21:00〜21:54
テストテレビ「週末サンプルショー」
```
